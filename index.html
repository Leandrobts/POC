PS4 Memory Corruption Tests (302) — MessageChannel transfer/clone invariants (rigorous)

• Objetivo: separar drop/atraso de entrega de sinais reais. ALERTA só dispara quando: sender detacha (byteLength=0) + PING OK + payload não chega até timeout.
• Se no PS4 houver drop apenas em pressure, isso tende a ser starvation/DoS do pipeline de tasks, não corrupção.
Dica prática: para “caçar janela”, aumente Timeout e reduza Delay. Para confirmar DoS, veja se ping_fail sobe junto.
